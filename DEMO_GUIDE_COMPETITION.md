## â“ **ANTICIPATED Q&A**

### **Q: Bagaimana jika user bypass aplikasi?**
**A:** 
"Pertanyaan bagus. OptiFokus punya beberapa layer:
1. System-level blocking (butuh permission device)
2. Adaptive difficulty - semakin bypass, quiz makin susah
3. Loss aversion warning - remind user tentang streak yang hilang
4. Social accountability - teman lihat jika user tidak konsisten

Kombinasi ini membuat bypass secara psikologis less rewarding."

### **Q: Quiz generation-nya pakai AI apa?**
**A:**
"Kami menggunakan NLP (Natural Language Processing) untuk extract key concepts
dari materi, lalu generate multiple choice questions dengan distractor answers
yang masuk akal. For MVP, kami pakai [sebutkan model jika ada, atau] 
kombinasi rule-based dan ML model yang kami train sendiri dari dataset
soal-soal kuliah."

### **Q: Berapa lama development-nya?**
**A:**
"Development dimulai [bulan] 2025, total 4 bulan hingga MVP saat ini.
2 bulan pertama research dan prototyping,
2 bulan berikutnya development dan testing dengan 50 mahasiswa."

### **Q: Target market dan strategi monetisasi?**
**A:**
"Target primer: 8 juta mahasiswa Indonesia.
Secondary: 15 juta pelajar SMA/SMK.

Monetisasi:
- Freemium: Basic features gratis (attract users)

partnership dengan bimbel dan universitas."

### **Q: Privacy dan data user?**
**A:**
"Privacy adalah prioritas. OptiFokus pakai local-first architecture:

- Analytics data di-anonymize

- Comply dengan UU PDP Indonesia
- User bisa delete semua data kapan saja"

### **Q: Competitive advantage vs aplikasi blocker lain?**
**A:**
"Aplikasi blocker existing seperti Freedom, Cold Turkey, dll hanya block apps.
User bisa dengan mudah uninstall atau bypass.

OptiFokus different karena:
1. UBCQ - Pertama di Indonesia, quiz dari materi sendiri
2. Gamified learning - Bukan forcing, tapi motivating
3. Adaptive system - Belajar dari behavior user
4. Social accountability - Peer pressure positif
5. Comprehensive - 9 features, bukan just blocker

Kami bukan just blocker, kami learning companion."

### **Q: Scalability? Bisa handle berapa user?**
**A:**
"Architecture kami dirancang scalable:
- Frontend: React PWA, bisa di-deploy ke CDN
- Backend: Microservices dengan Kubernetes (jika pakai backend)
- Database: Sharded untuk horizontal scaling

Conservative estimate: 100K concurrent users tanpa issue.
Jika perlu scale lebih, tinggal tambah resources (cloud-native)."
